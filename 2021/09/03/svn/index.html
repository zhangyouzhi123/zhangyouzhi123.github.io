检查是否有安装历史版本 rpm -qa|grep subvision

我们这里使用centos6.5版本默认没有安装，所以不需要卸载，如果有安装的话，需要先卸载。

安装svn相关应用 yum install subversion

检查安装是否成功

检查服务器版本 svnserve –version

检查客户端版本 svn –version

安装成功后，已经可以作为SVN客户端使用，我们可以先检出一个已经存在的SVN项目测试下我们的客户端是否可以使用,接下来我们来测试SVN客户端的使用是否正常。

常用命令我们可以通过 svn –help来查看

创建目录用于检出文件 mkdir ~/svnOutDir

\00001. 检出文件

· 完整写法 svn checkout SVN检出路径 –username=用户名 –password=密码 /path/to/检出路径

· 简略写法 svn checkout SVN检出路径

yes 保存密码 no 不保存密码

项目检出成功，证明我们的客户端是没有问题的，其他svn相关命令可以参考svn命令的使用。

SVN服务器的使用

如果我们需要让我们的Linux服务器本身作为svn的服务端，我们需要进行如下配置。

创建仓库目录用于存放项目 mkdir ~/svnrepos

在仓库目录中新建仓库 svnadmin create ~/svnrepos/proj1

\00001. 修改仓库中的配置文件

查看配置文件 ls ~/svnrepos/proj1/conf

修改password 添加用户以及用户 密码 vi ~/svnrepos/proj1/conf/passwd

修改authz 配置用户权限 vi ~/svnrepos/proj1/conf/authz

添加三行配置信息：

developers = ouyang
[proj1:/]
@developers = rw

修改svnserve.conf配置svn服务器 vi ~/svnrepos/proj1/conf/svnserve.conf

到此，我们的svn配置完成，现在我们开始启动服务 svnserve -d -r ~/svnrepos

其中 -d 表示启动守护进程 -r 表示svn根目录

我们在本地测试一下检出 svn checkout svn://localhost/proj1 –username=ouyang –password=123456 ~/proj1

这里表示使用用户ouyang 密码123456 检出本地proj1仓库到 家目录下的proj1文件夹下

本地检出正常，如果需要配置外部访问，我们需要开启端口3690

开启3690端口 iptables -I INPUT -p tcp –dport 3690 -j ACCEPT

保存配置 /etc/rc.d/init.d/iptables save
